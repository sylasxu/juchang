# 微信聊天工具模式 (Chat Tool Mode)

> **版本**: Phase 2
> **状态**: 规划中

## 概述

用群聊替代推送，彻底解决微信订阅消息的限制问题。

## 核心价值

| 订阅消息的坑 | 聊天工具模式 |
|-------------|-------------|
| 用户要主动点「允许」 | 不需要授权 |
| 一次授权只能发一条 | 没有次数限制 |
| 内容受限 | 自定义内容 |
| 触达率低 | 所有群成员都能看到 |

## 通知策略

| 场景 | 旧策略 | 新策略 |
|------|--------|--------|
| 有人报名 | 订阅消息给群主 | 群里自动冒泡「alex 已参与」|
| 活动快开始 | 订阅消息提醒 | 卡片状态变「即将开始」|
| 催人报名 | 没有能力 | `wx.notifyGroupMembers` |

## 技术约束

- 渲染引擎: Skyline
- 分包类型: 独立分包
- 体积限制: ≤ 500KB
- 基础库: ≥ 3.7.8

## 核心 API

| API | 用途 |
|-----|------|
| `wx.openChatTool` | 进入聊天工具模式 |
| `wx.getChatToolInfo` | 获取绑定群聊信息 |
| `wx.shareAppMessageToGroup` | 发送卡片到群聊 |
| `wx.notifyGroupMembers` | @ 群成员提醒 |

## 实施优先级

| 阶段 | 内容 |
|------|------|
| P0 | 基础分包 + 进入/退出流程 |
| P1 | 动态消息 (卡片实时更新) |
| P2 | 群成员选择 + @ 提醒 |
