# 聚场 (JuChang) 产品需求文档

> **版本**: v4.2 | **更新**: 2026-01
> **定位**: Personal Social Agent (个人社交代理人)
> **Slogan**: 想怎么玩？跟小聚说说。

---

## 1. 产品定位

### 1.1 我们是什么

聚场是一个 **AI 组局助理**，用户通过对话告诉小聚想玩什么，AI 自动完成活动创建、搭子匹配、报名管理。

| 维度 | 传统工具 | 聚场 (Agent) |
|------|---------|-------------|
| 交互 | 填表单、点按钮 | 说话，AI 理解并执行 |
| 界面 | 静态 UI | Generative UI，动态生成 |
| 服务 | 只服务群主 | 服务每一个人 |
| 记忆 | 用完即走 | 记住你，下次更懂你 |

### 1.2 我们解决什么

| 旧世界痛点 | 聚场解法 |
|-----------|---------|
| 群里喊话 5 分钟被淹没 | AI 生成精美活动邀请函 |
| 陌生人组局不靠谱 | 精致感本身就是信任背书 |
| 发起人像复读机 | 系统自动统计，不用复读 |

### 1.3 竞品差异

**粗门 vs 聚场**:
- 粗门 = 活动电商平台，服务 B 端俱乐部
- 聚场 = AI 组局助理，服务普通群主

| 场景 | 谁占优势 |
|------|---------|
| 商业活动 (百人徒步) | 粗门 |
| 朋友聚餐 (火锅、烧烤) | 聚场 |
| 临时起意 (今晚喝一杯) | 聚场 |

首页是对话，不是列表。用户说话，AI 执行。

| 查询 | 我的活动 | getMyActivities |

## 3. 找搭子 (Broker Mode)

用户想参加某类活动但不想自己组局，Agent 作为"高级经纪人"帮用户找搭子。

### 3.1 核心流程

```
用户: "想吃火锅，谁组我就去"
  ↓
Agent 进入 Broker Mode，结构化追问
  ↓
用户: "1A 2A 3A" (今晚/AA/不喝酒)
  ↓
Agent 调用 createPartnerIntent
  ↓
系统匹配 → 创建 Match → Temp_Organizer 确认 → 转为 Activity
```

### 3.2 结构化追问

```
好的，帮你找火锅搭子！🍲 请确认：

1. ⏰ 时间？ A: 今晚  B: 明天  C: 周末
2. 💰 费用？ A: AA制  B: 有人请客也行
3. 🎯 要求？ A: 不喝酒  B: 安静点  C: 女生友好  D: 没有

回复：**1A 2A 3AD**
```

### 3.3 匹配规则

- 活动类型完全匹配
- 位置在 3km 内
- 无 tag 冲突 (AA ↔ Treat, NoAlcohol ↔ Drinking)
- 匹配分数 ≥ 80%

### 3.4 生命周期

- Intent: active → matched/expired(24h)/cancelled
- Match: pending → confirmed/expired(6h)/cancelled

## 4. AI 运营能力

### 4.1 用户画像

小聚记住用户偏好：活动类型、常去地点、时间偏好。

### 4.2 内容审核

AI 检测活动内容风险：敏感词、联系方式泄露、风险评分。

### 4.3 异常检测

- 批量创建：24h 内创建 >10 个活动
- 频繁取消：7d 内取消 >5 次

### 5.1 微信聊天工具模式

## 7. 成功指标

### 北极星: J2C 转化率

```
J2C = 先参局后组局的用户数 / 历史总参与者数
```

| J2C | 含义 |
|-----|------|
| < 1% | 单纯工具 |
| 1-5% | **PMF 达标** |
| > 5% | 病毒式增长 |

### 核心指标

| 指标 | 达标线 |
|------|--------|
| 草稿→发布转化率 | ≥ 60% |
| 活动成局率 | ≥ 50% |
| 周留存率 | ≥ 15% |
