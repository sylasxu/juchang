# 聚场 (Juchang) 产品需求文档 (Lean MVP)

> **核心理念**：Map Copilot (地图副驾) —— 用 AI 接住用户的自然语言，把"群聊的流"变成"地图的桩"
> 
> **产品定位**：懂你的重庆本地搭子助手

---

## 0. 产品定位与核心价值

### 0.1 我们解决什么问题？

**核心矛盾**：用户想组局/找搭子，但现有渠道（微信群/直播间）存在"工具的冷漠"与"社交的热闹"之间的矛盾。

#### 旧世界的痛点

| 场景 | 微信群的问题 | 用户心理 |
|------|-------------|----------|
| 发起组局 | "周五观音桥火锅，有人吗？" → 死一般的寂静，消息被顶走 | 是不是我没号召力？是不是大家不想理我？ |
| 信息管理 | 刚才还在说火锅，马上有人插嘴说球赛，信息瞬间被冲走 | 想找刚才那个人？翻聊天记录翻到死 |
| 操作门槛 | 在群里打字多快啊，来你这还要选时间、选地点、填标题 | 太累了，算了不发了 |
| 社交面子 | 到处发"求求了，差一个人，救救孩子" | 显得很卑微，不想这样求人 |

### 0.2 聚场的解决方案

**核心理念**：微信群是"前线"（流量大但效率低），聚场是"指挥部/大后方"（有地图沙盘、有参谋长、有军规）

```
用户去前线（微信群）吆喝 → 把人拉回指挥部（聚场）确认、管理、沉淀
```

### 0.3 四大核心优势

#### 🎯 响应感 —— 治愈"无人回应的焦虑"

| 旧世界（微信群） | 新世界（聚场） |
|-----------------|---------------|
| 发消息后死一般的寂静 | AI 秒回，0.5s 内开始响应 |
| 运气好有人回表情包，运气不好消息被顶走 | 流式反馈：文字逐字显示，地图同步飞向目标位置 |
| 用户心里很慌：是不是没人理我？ | 不管有没有真人，系统秒回了你 |

```
用户说："周五观音桥火锅，3缺1"

CUI 副驾反馈：
├── 0.5s："收到，正在锁定观音桥..."（地图飞过去，视觉反馈）
├── 1.0s："正在检索..."（文字逐字跳动，情绪反馈）
└── 1.5s："附近暂无，但我帮你把局组好了，一键发布吗？"（行动反馈）
```

#### 📍 秩序感 —— 把"混乱的流"变成"确定的桩"

| 旧世界（微信群） | 新世界（聚场） |
|-----------------|---------------|
| 信息是流（Flow），瞬间被冲走 | 信息是桩（Pin），24小时内雷打不动 |
| 想找刚才那个人？翻聊天记录翻到死 | 橙色 Pin 插在观音桥地图上，一目了然 |
| 在群里吼一声是"一次性的" | 在聚场发一个是"持久的" |

#### ⚡ 零门槛 —— 不改变习惯，只提升效率

| 旧世界（填表单） | 新世界（AI 输入栏） |
|-----------------|-------------------|
| 选时间、选地点、填标题，太累了 | 在群里怎么说话，在这里就怎么说话 |
| 需要学习新操作 | 直接把群里的乱七八糟复制过来，自动变整齐 |
| 结构化输入 | 自然语言输入 → 结构化输出 |

#### 👑 面子工程 —— 从"乞讨式组局"到"海报式发令"

| 旧世界（微信群） | 新世界（聚场） |
|-----------------|---------------|
| "求求了，差一个人，救救孩子" | 生成精美海报："🔥急招 \| 缺1人 \| 倒计时2小时" |
| 显得很卑微 | 往群里一丢，这是"发通告"，不是"求人" |
| 没有掌控感 | 用这个 App 组局显得自己很专业、很酷 |

### 0.4 产品心法

**核心洞察**：在陌生人社交里，"有人（哪怕是AI）回应我"本身就是一种极其重要的情绪价值。

**设计哲学**：
```
以前：输入 -> 黑盒 -> 结果（这是工具）
现在：输入 -> AI 思考 -> 地图响应 -> 给出建议（这是伙伴）
```

## 1. 商业模式与权益 (Economics)

### 1.1 核心策略：微交易 (Micro-Transactions)

**设计哲学**：
- **流量免费**：搜索、对话、看地图完全免费，确保用户流失率最低
- **发布克制**：限制普通用户建局数量，保证地图质量
- **焦虑变现**：只卖"流量曝光"（Boost/Pin+），这是组局者最痛的点

### 1.2 免费权益 (Free Tier)

| 功能 | 免费额度 | 说明 |
|------|----------|------|
| AI 副驾 (搜索/解析) | 50次/天 | 几乎无感，鼓励用户多说话、多探索 |
| 发布活动 | 3次/天 | 防止垃圾广告，满足正常需求 |
| 报名参与 | ✅ 无限 | 核心功能 |
| 群聊沟通 | ✅ 无限 | 核心功能 |
| 履约/评价 | ✅ 无限 | 核心功能 |

### 1.3 付费增值服务 (Pay-per-Use)

只保留高转化、低开发成本的工具：

| 服务 | 价格 | 场景与痛点 | 利润率 |
|------|------|------------|--------|
| 🚀 **强力召唤 (Boost)** | ¥3/次 | "急缺人"。3km 内推送 + 地图 Pin 闪烁 | ~95% |
| 👑 **黄金置顶 (Pin+)** | ¥5/次 | "求关注"。地图 Pin 变大变金 (24h) | ~95% |

> MVP 阶段砍掉：会员订阅、风控报告、优先入场券、AI 额度包

### 1.4 盈利模型

**假设**：DAU 1000

```
成本预估：
- 服务器: ¥30/天
- AI 搜索: 5000次 × ¥0.001 = ¥5/天 (低价模型，成本极低)
- 其他: ¥15/天
成本合计: ¥50/天

收入预估：
- Boost/Pin+: 30次 × ¥4 (平均) = ¥120/天

日利润: ¥70 (利润率 58%)
```

**结论**：只要每天有 30 个人付费推广，就能覆盖所有成本并盈利。

## 2. 核心功能：CUI 地图副驾 (Map Copilot)

### 2.1 底部导航结构

**3 Tab + AI 输入栏设计**：
- **Tab 1**: 首页 (Home) - 地图 + AI 输入栏综合页
- **Tab 2**: 消息 (Connect) - 社交连接
- **Tab 3**: 我 (Me) - 个人中心
- **AI 输入栏**: 底部常驻悬浮栏 - 全能 CUI 入口

### 2.2 入口：AI 输入栏

```
┌─────────────────────────────────────────────────────────┐
│                      地图区域                           │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐   │
│  │ 🤖  本周想玩什么...                        🎙️ │   │  ← AI 输入栏
│  └─────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────┤
│     [首页]        [消息]        [我]                    │  ← Tabbar
└─────────────────────────────────────────────────────────┘
```

- **位置**：底部 Tabbar 上方悬浮
- **外观**：类似灵动岛的黑色长条，左侧 AI 图标，中间提示文案，右侧语音按钮
- **功能**：整合"搜索"与"创建"为单一入口

### 2.3 交互流程：从意图到结果

**用户输入**："明晚观音桥打麻将，3缺1" (或粘贴群消息)

**AI 副驾反馈 (流式响应)**：

```
┌───────────────────────────────────────┐
│ 🤖 CUI 副驾面板                        │
├───────────────────────────────────────┤
│                                       │
│ 思考态："收到，正在定位观音桥..."      │
│ (地图同步飞向目标点)                   │
│                                       │
│ 搜索态："正在检索附近的麻将局..."      │
│ (地图 Pin 动态过滤)                    │
│                                       │
│ 结果态 (双选卡片)：                    │
│ ┌─────────────────────────────────┐   │
│ │ A. 🔍 发现 2 个局，去看看？     │   │
│ └─────────────────────────────────┘   │
│ ┌─────────────────────────────────┐   │
│ │ B. 🀄️ 麻将局·3缺1              │   │
│ │    📍 观音桥 · 明晚             │   │
│ │    [🚀 立即发布]                │   │
│ └─────────────────────────────────┘   │
│                                       │
└───────────────────────────────────────┘
```

### 2.4 结果卡片设计

**创建草稿卡片 (Draft Card)**：
- 标题：麻将局·3缺1 (AI 自动提取)
- 时间：明晚 19:00
- 地点：观音桥 (已定位)
- 行动点：[🚀 立即发布] (点击即上地图)

## 3. 地图与 LBS (Map Logic)

### 3.1 渲染逻辑

| 类型 | 颜色 | 说明 |
|------|------|------|
| 真实活动 | 🟠 橙色 Pin | 用户发布的活动 |
| 幽灵锚点 | 🟢 绿色虚线 Pin | 运营投放的需求引导 |
| 我的位置 | 🔵 蓝色光点 | 当前用户位置 |
| Boost 活动 | 🟠 橙色闪烁 | 付费推广，带"🔥急招"标签 |
| Pin+ 活动 | 🟡 金色 1.5x | 付费置顶，带光晕动效 |

### 3.2 幽灵锚点 (Ghost Anchors)

**设计原则**：严禁伪装成真实活动

**交互逻辑**：
```
点击绿色锚点 → 直接唤起 AI 输入栏 → 复用"AI 创建"流程
预填：类型=锚点建议类型，位置=当前锚点
```

**锚点文案示例**：
- "这里缺一个火锅局 🍲"
- "有人想在这玩飞盘 🥏"
- "周末这里适合野餐 🧺"

### 3.3 重庆地形适配

**必填项**：在确认发布时，强制要求填写 `location_hint` (位置备注)

**选项**：
- 地面入口
- 地下通道进
- XX楼平台
- 其他：[自定义]

## 4. 社交与履约 (Trust & Connect)

### 4.1 简化的靠谱度

**取消**：复杂的百分比计算
**保留**：徽章展示

| 履约率 | 展示 |
|--------|------|
| > 90% | 🏅 超靠谱 |
| > 80% | ✓ 靠谱 |
| 新人/低履约 | 🆕 新人 |

### 4.2 履约流程：静默生效

```
活动结束
    │
    ▼
发起人收到确认通知
    │
    ├── 24h内操作 → 标记到场/未到场
    │
    └── 超时未操作 → 默认全员履约 ✅
```

**被标记"未到场"者**：
- 收到推送通知，内含 [我到场了] 按钮
- 点击即申诉成功（MVP阶段：视为争议，双方均不扣分）

### 4.3 流量闭环 (Growth)

**群接龙识别**：AI 输入栏自动读取剪贴板，解析群接龙文本

**微信原生分享**：发布成功后，引导用户使用微信原生分享功能，分享卡片包含活动标题、时间、缺人数等关键信息，点击直接打开活动详情页。

## 5. 技术实现关键点 (Tech Specs)

### 5.1 AI 策略 (Cost Control)

**模型选择**：
- **搜索/对话**：使用 DeepSeek-V3 / Haiku 等超低价模型，速度快，便宜
- **复杂解析**：仅在"确认发布"时做一次高质量校验

**防刷机制**：
- 前端做 Debounce (防抖)，用户停止输入 500ms 后才请求 AI

**流式响应 API**：
```typescript
// POST /ai/parse (支持 SSE)
// 流式响应事件
{ event: "thinking", data: { message: "正在解析..." } }
{ event: "location", data: { name: "观音桥", coords: [106.5, 29.5] } }
{ event: "searching", data: { message: "搜索附近活动...", progress: 50 } }
{ event: "result", data: { activities: [...], draft?: { ... } } }
{ event: "done" }
```

### 5.2 数据库精简 (Schema)

仅保留核心表：
- `users` (含 Auth, Profile, Quota)
- `activities` (含 Ghost, ChatStatus, LocationHint)
- `participants` (含 Fulfillment)
- `transactions` (含 Payment)
- `chat_messages` (活动群聊)

### 5.3 懒注册

- 游客可浏览地图、查看活动详情
- 以下操作触发登录：创建活动、报名、查看群聊

## 6. 开发路线图 (Roadmap)

### Phase 1: 最小闭环 (2 Weeks)

- **地图 + AI 输入栏**：实现"说话控制地图"的核心体验
- **AI 解析**：接入低价模型，跑通"文本 → 结构化草稿"
- **发布与展示**：让活动能真实地插在地图上
- **基础社交**：报名、群聊、履约确认

### Phase 2: 流量与支付 (1 Week)

- **分享图生成**：Canvas 绘图
- **微信支付**：Boost / Pin+ 功能
- **扫码落地**：scene 参数解析

## 附录：MVP 砍掉的功能

| 功能 | 砍掉原因 |
|------|----------|
| 会员订阅 (Pro) | MVP 阶段用户不会包月，订阅制开发复杂 |
| AI 深度风控报告 | Day 1 没数据，场景过重，Token 成本高 |
| 优先入场券 (Fast Pass) | 冷启动期缺人，不是人太多挤不进去 |
| AI 额度包 | 搜索是流量入口，不能收费阻拦 |

**这些功能在用户量起来后可以重新考虑。**
