> **版本**: v4.0 | **更新**: 2025-01
> **定位**: Personal Social Agent (个人社交代理人)
> **Slogan**: 想怎么玩？跟小聚说说。

---

## 1. 产品哲学

### 1.1 Agent vs Tool

| 维度 | 传统工具 | 聚场 (Agent) |
|------|---------|-------------|
| 交互模式 | 填表单、点按钮 | 说话，AI 理解并执行 |
| 界面生成 | 静态 UI | Generative UI，根据意图动态生成 |
| 服务对象 | 只服务群主 | 服务每一个人 |
| 用户关系 | 用完即走 | 记住你，下次更懂你 |

### 1.2 Chat-First 架构

首页是对话，不是列表/地图。这不是 UI 风格，是产品定位：
- 货架模式 = 你是顾客，我是商城
- 对话模式 = 你是老板，我是秘书

### 1.3 我们解决什么

**旧世界痛点**:
- 信息流失: 群里喊话 5 分钟被淹没
- 信任缺失: 陌生人组局不靠谱
- 沟通成本: 发起人像复读机

**聚场解法**:
- 变"流"为"桩": AI 生成精美活动邀请函
- 建立信任: 精致感本身就是信任背书
- 自动化管理: 系统自动统计，不用复读

## 3. 竞品差异化

### 3.1 粗门 vs 聚场

| 维度 | 粗门 | 聚场 |
|------|------|------|
| 定义 | 活动电商平台 | AI 组局效能工具 |
| 发起人 | B端/俱乐部主理人 | 普通群主 |
| 流量逻辑 | 公域分发 | 私域流转 |
| 操作成本 | 极高 (填表、传海报) | 极低 (一句话) |

### 3.2 场景隔离

| 场景 | 谁占优势 |
|------|---------|
| 商业活动 (百人徒步) | 🔴 粗门 |
| 朋友聚餐 (火锅、烧烤) | 🟢 聚场 |
| 临时起意 (今晚喝一杯) | � 聚场 |
| 半熟人拼车/拼单 | 🟢 聚场 |

**一句话差异化**: "粗门是给俱乐部卖票的电商平台；聚场是你的 AI 活动助理。"

## 5. 成功指标

### 5.1 北极星: J2C 转化率

```
J2C = 先参局后组局的用户数 / 历史总参与者数
```

| J2C | 含义 |
|-----|------|
| < 1% | 单纯工具，只有群主在用 |
| 1-5% | **PMF 达标线** |
| > 5% | 病毒式增长 |

### 5.2 核心指标

| 指标 | 达标线 |
|------|--------|
| 草稿→发布转化率 | ≥ 60% |
| 活动成局率 | ≥ 50% |
| 周留存率 | ≥ 15% |
